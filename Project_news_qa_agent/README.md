# AI News Q&A Agent

관심 주제에 대한 최근 뉴스를 수집하고, **수집된 기사 범위 안에서 근거를 들어 답변하는 RAG 기반 뉴스 Q&A 에이전트**입니다.

프로그램 시작 시 뉴스를 한 번 수집하고 FAISS 인메모리 벡터스토어를 구축합니다. 이후 질문은 추가 크롤링 없이 저장된 기사에서 관련 근거를 검색해 처리합니다.

```text
관심 분야를 입력하세요: 제논
검증 키워드(콤마 구분): AI기업
기간(일): 30
...
질문> 제논이 최근 집중하는 사업은 뭐야?
```

## 프로젝트 목표

일반적인 LLM은 최신 이슈에 답하기 어렵거나, 충분한 근거 없이 사실처럼 보이는 답변을 생성할 수 있습니다.

이 프로젝트는 사용자가 지정한 기간의 뉴스를 실시간으로 수집하고, 해당 기사만 답변 근거로 사용하도록 설계했습니다. 답변에는 `(기사 N)` 형식으로 출처를 표시하며, 기사 데이터는 별도 데이터베이스에 저장하지 않고 세션 동안만 메모리에 유지합니다.

## 핵심 기능

### 검색 주제와 검증 키워드 분리

사용자는 뉴스 검색에 사용할 관심 주제와 기사 관련성을 판정할 검증 키워드를 별도로 입력합니다.

```text
관심 분야: 제논
검증 키워드: AI기업
```

관심 분야는 뉴스 검색어 생성에 사용하고, 검증 키워드는 수집된 기사가 사용자의 의도와 실제로 관련 있는지 판정하는 기준으로 사용합니다. 이를 통해 동음이의어나 문맥이 다른 기사를 줄일 수 있습니다.

### 검색 키워드 자동 생성

사용자가 입력한 관심 분야를 그대로 검색하면 약어, 영문명, 정식 명칭 등의 표기 차이로 일부 기사를 놓칠 수 있습니다.

LLM은 관심 분야에서 다음 기준으로 최대 4개의 뉴스 검색 키워드를 생성합니다.

* 입력에 포함된 고유명사를 원문 그대로 1순위에 유지
* 확실한 경우에만 정식 명칭, 약어, 영문 표기 추가
* 동음이의어 가능성이 있는 경우 업종이나 맥락어를 결합한 키워드 추가
* `AI 기업`, `최근 이슈`, `산업 동향`처럼 지나치게 넓은 검색어 제외

예시는 다음과 같습니다.

```text
입력: 서울여대
출력: ["서울여대", "서울여자대학교", "SWU"]
```

LLM이 JSON 형식을 지키지 못하면 사용자가 입력한 원문을 검색 키워드로 사용하는 폴백 로직을 적용합니다.

수집 기간은 LLM이 추론하지 않으며, 사용자가 입력한 숫자를 그대로 사용합니다.

### 하이브리드 관련성 필터

수집된 기사는 임베딩 유사도와 LLM 판정을 결합해 관련성을 검사합니다.

먼저 임베딩으로 명확히 관련 있거나 무관한 기사를 빠르게 분류하고, 판단이 애매한 기사만 LLM이 추가로 판정합니다. 이를 통해 모든 기사를 LLM으로 평가하는 방식보다 비용과 처리 시간을 줄이면서 의미 기반 필터링을 유지합니다.

### 의미 기반 중복 제거

제목이나 문장이 다르더라도 내용이 사실상 동일한 기사는 임베딩 코사인 유사도를 기준으로 하나만 남깁니다.

관련성 필터에서 생성한 기사 임베딩을 중복 제거에도 재사용해 불필요한 임베딩 API 호출을 줄였습니다.

### HyDE 기반 검색

질문과 기사에 사용된 표현이 다르면 질문을 그대로 임베딩하는 방식으로 관련 기사를 찾지 못할 수 있습니다.

HyDE는 질문에 대한 답이 포함되어 있을 법한 가상의 기사 문장을 생성한 뒤, 해당 문장을 임베딩해 실제 기사를 검색합니다. 이를 통해 인물, 사업, 행사, 제품 설명처럼 질문과 기사 표현이 직접 일치하지 않는 사례의 검색 성능을 개선했습니다.

### 임계값 기반 기사 검색

질문마다 항상 같은 개수의 기사를 반환하지 않습니다.

질문과 기사의 코사인 유사도가 `RELEVANCE_SIMILARITY_THRESHOLD` 이상인 기사만 출처 후보로 사용하며, `TOP_K_ARTICLES`는 반환할 수 있는 최대 기사 수로만 동작합니다.

### 근거 인라인 인용

답변은 자연스러운 문단으로 생성하며, 기사로 확인된 내용에는 다음과 같이 출처를 표시합니다.

```text
제논은 최근 액셔너블 AI와 기업용 AI 에이전트 사업을 확대하고 있습니다. (기사 2)
```

수집된 기사에서 관련 근거를 찾지 못한 경우에는 기사에 없는 내용을 기사 근거처럼 표현하지 않습니다.

## 동작 흐름

1. 관심 분야, 검증 키워드, 수집 기간을 입력받습니다.
2. LLM이 뉴스 검색용 키워드 후보를 생성합니다.
3. 네이버 뉴스 검색 API로 기사 메타데이터를 수집합니다.
4. `aiohttp`와 `BeautifulSoup`으로 기사 본문을 크롤링합니다.
5. 기사 본문을 임베딩합니다.
6. 임베딩과 LLM을 이용해 관련 기사를 필터링합니다.
7. 의미적으로 중복된 기사를 제거합니다.
8. 정제된 기사로 FAISS 인메모리 벡터스토어를 구축합니다.
9. 질문을 HyDE 문장으로 변환하고 관련 기사를 검색합니다.
10. 유사도 임계값을 통과한 기사만 사용해 답변과 출처를 생성합니다.

## 기술 스택

Python · LangChain LCEL · OpenAI · FAISS · HyDE · aiohttp · BeautifulSoup · Naver Cloud Platform News Search API

사용 모델:

* Chat model: `gpt-4o-mini`
* Embedding model: `text-embedding-3-small`

## 폴더 구조

```text
main.py                       # 뉴스 수집 세션 및 질문 루프
build_labeled_set.py          # 관련성 평가용 기사 라벨링
eval_filter.py                # 관련성 필터 성능 평가
eval_retrieval.py             # HyDE 검색 성능 평가
config.py                     # 환경변수 및 튜닝 설정

app/
  schemas.py                  # 데이터 모델
  query_understanding.py      # 뉴스 검색 키워드 생성
  news_search.py              # 네이버 뉴스 검색 API
  crawler.py                  # 기사 본문 크롤링
  embeddings.py               # 공유 임베딩 클라이언트
  relevance_filter.py         # 임베딩 + LLM 관련성 필터
  deduplication.py            # 의미 기반 중복 제거
  vector_store.py             # FAISS 구축 및 HyDE 검색
  rag_chain.py                # 근거 기반 답변 생성
```

## 시작하기

### 1. 가상환경 생성

```bash
python -m venv venv
```

Windows:

```bash
venv\Scripts\activate
```

Linux 또는 macOS:

```bash
source venv/bin/activate
```

### 2. 패키지 설치

```bash
pip install -r requirements.txt
```

### 3. 환경변수 설정

`env.example.txt`를 복사해 `.env` 파일을 생성한 뒤 다음 값을 입력합니다.

```text
OPENAI_API_KEY=
NAVER_CLIENT_ID=
NAVER_CLIENT_SECRET=
```

### 4. 실행

```bash
python main.py
```

네이버 뉴스 API는 Naver Cloud Platform의 API Gateway 인증 방식을 사용합니다. 기존 네이버 개발자센터 API와 요청 주소 및 인증 헤더가 다르므로 주의해야 합니다.

## 환경변수 및 설정값

| 변수                               |                      기본값 | 설명                            |
| -------------------------------- | -----------------------: | ----------------------------- |
| `OPENAI_API_KEY`                 |                       필수 | OpenAI API 키                  |
| `OPENAI_CHAT_MODEL`              |            `gpt-4o-mini` | 키워드 생성, 판정, 답변 생성 모델          |
| `OPENAI_EMBEDDING_MODEL`         | `text-embedding-3-small` | 기사 및 질문 임베딩 모델                |
| `NAVER_CLIENT_ID`                |                       필수 | NCP API Gateway Client ID     |
| `NAVER_CLIENT_SECRET`            |                       필수 | NCP API Gateway Client Secret |
| `DEDUP_SIMILARITY_THRESHOLD`     |                   `0.90` | 중복 기사로 판단할 코사인 유사도 기준         |
| `TOP_K_ARTICLES`                 |                      `5` | 답변 근거로 반환할 최대 기사 수            |
| `MAX_CRAWL_ARTICLES`             |                    `100` | 한 세션에서 크롤링할 최대 기사 수           |
| `CRAWL_TIMEOUT_SECONDS`          |                      `8` | 기사 한 건의 크롤링 제한 시간             |
| `RELEVANCE_SIMILARITY_THRESHOLD` |                   `0.30` | 질문과 기사 사이의 최소 검색 유사도          |

프로그램 시작 시 필수 API 키가 누락되었는지 검사하며, 누락된 값이 있으면 실행을 중단하고 해당 환경변수를 안내합니다.

## 관련성 필터 평가

`build_labeled_set.py`를 이용해 실제 기사의 관련 여부를 직접 라벨링하고, `eval_filter.py`로 필터 적용 전후 성능을 비교할 수 있습니다.

```bash
python build_labeled_set.py "제논" "AI기업" 20
python eval_filter.py
```

`제논 + AI기업` 조건으로 수집한 기사 97건을 평가한 결과는 다음과 같습니다.

| 지표        | 필터 적용 전 | 필터 적용 후 |
| --------- | ------: | ------: |
| Precision |   0.309 |   0.625 |
| Recall    |   1.000 |   1.000 |
| F1        |   0.472 |   0.769 |
| Accuracy  |   0.309 |   0.814 |

관련 기사 Recall을 유지하면서 Precision과 Accuracy가 개선되어, 무관 기사 제거 효과를 확인했습니다.

## HyDE 검색 평가

`eval_retrieval.py`는 질문을 그대로 임베딩하는 방식과 HyDE를 적용한 방식의 Retrieval Hit Rate를 비교합니다.

```bash
python eval_retrieval.py "제논" "AI기업" 30
```

실제 대화에서 검색에 실패했던 질문과 기존 검색 성공 질문을 포함한 골든 질문 5개를 평가한 결과입니다.

| 방식            | 검색 성공률 |
| ------------- | -----: |
| V0: 질문 직접 임베딩 |  20.0% |
| V1: HyDE 적용   |  80.0% |

HyDE 적용 후 검색 성공률이 `20%`에서 `80%`로 증가했으며, 질문과 기사 표현이 직접 일치하지 않는 사례에서 개선을 확인했습니다.

현재 평가는 5개 질문을 기준으로 한 소규모 실험이므로, 향후 다양한 질문 유형으로 골든셋을 확장할 예정입니다.

## 알려진 이슈

### `httpx proxies` 오류

OpenAI 패키지와 일부 `httpx` 버전의 호환성 문제로 발생할 수 있습니다.

```text
httpx==0.27.2
```

### 네이버 API 401 오류

기존 네이버 개발자센터 API와 NCP API Gateway 방식은 인증 헤더가 다릅니다. 현재 프로젝트는 `X-NCP-APIGW-*` 헤더를 사용하는 NCP 방식으로 구현되어 있습니다.

### 콘솔 한글 입력 오류

PyCharm 콘솔과 한글 IME 조합에서 입력이 누락되거나 깨질 수 있습니다. UTF-8 설정과 재입력 처리를 적용했으며, 문제가 지속되면 실행 인자로 값을 전달하는 방식을 사용할 수 있습니다.

```bash
python eval_retrieval.py "제논" "AI기업" 30
```

### 실행 시점에 따른 결과 변화

뉴스 검색 결과는 기사 게시, 삭제, 검색 순위, 크롤링 성공 여부에 따라 달라질 수 있습니다. 평가 결과를 비교할 때는 실행 날짜와 입력 조건을 함께 기록하는 것이 좋습니다.

## 한계 및 개선 방향

* 언론사별 HTML 구조에 따라 기사 본문 크롤링이 실패할 수 있습니다.
* 관련성 및 중복 제거 임계값은 주제에 따라 재조정이 필요할 수 있습니다.
* HyDE가 부정확한 가상 문장을 생성하면 검색 성능이 저하될 수 있습니다.
* 현재 기사와 벡터스토어는 세션 종료 시 삭제됩니다.
* 하나의 실행 세션에서는 하나의 주제만 사용할 수 있습니다.

향후에는 다음 기능을 추가할 예정입니다.

* 원 질문과 HyDE 임베딩을 결합한 앙상블 검색
* BM25와 벡터 검색을 결합한 하이브리드 검색
* 답변 Faithfulness와 Correctness 자동 평가
* 출처 신뢰도 및 낚시성 기사 필터
* 세션 중 주제 전환
* LangGraph 기반 검색·검증 에이전트 분리
